# Booking Manicure Bot

Telegram-бот для записи на маникюр и другие услуги с inline-интерфейсом, клиентским меню и встроенной админ-панелью в Telegram.

Проект рассчитан на одного мастера на старте, но структура уже подготовлена к расширению на нескольких мастеров.

## Возможности

### Клиентская часть

- просмотр приветственного экрана
- inline-меню в Telegram
- просмотр контактов мастера
- выбор услуги
- выбор даты записи
- выбор свободного времени
- подтверждение записи
- просмотр своих ближайших записей
- просмотр подробной карточки записи
- отмена записи с подтверждением
- запрет отмены меньше чем за час до начала
- получение напоминаний за день и за час
- получение уведомлений, если администратор:
  - подтвердил запись
  - отменил запись
  - перенес запись

### Админ-панель

- вход через `/admin`
- доступ только для `ADMIN_IDS`
- управление услугами:
  - создание
  - включение и выключение
  - удаление
  - редактирование названия
  - редактирование цены
  - редактирование длительности
  - редактирование описания
- управление графиком:
  - общие часы работы
  - исключения по датам
  - выходные дни по отдельным датам
  - особые часы на конкретную дату
  - ручные блокировки времени
- управление записями:
  - просмотр ближайших записей
  - просмотр записей по выбранной дате
  - подтверждение заявок
  - отмена записи
  - перенос записи
  - завершение записи
- работа с клиентами через карточку записи:
  - заметка к клиенту
- настройка приветственного текста
- настройка контактов мастера:
  - имя
  - телефон
  - Telegram
  - Instagram
- переключение автоподтверждения записей
- уведомление админу о новой записи

## Как работает расписание

В проекте используется гибридная схема расписания:

- есть общие часы работы на каждый день
- поверх них можно задавать исключения по конкретным датам
- отдельно можно блокировать произвольные интервалы времени

Пример:

- общие часы: `09:00 - 20:00`
- на `20.03.2026` можно поставить выходной
- на `21.03.2026` можно задать свои часы `12:00 - 18:00`
- на любой день можно дополнительно заблокировать, например, `14:00 - 15:30`

Свободные слоты считаются с шагом `30 минут`, но длительность услуги учитывается полностью. Если услуга длится `3 часа 30 минут`, бот ищет непрерывное свободное окно нужной длины.

## Установка

### 1. Создай виртуальное окружение

```powershell
python -m venv .venv
```

### 2. Активируй его

```powershell
.\.venv\Scripts\Activate.ps1
```

### 3. Установи зависимости

```powershell
.\.venv\Scripts\pip.exe install aiogram sqlalchemy aiosqlite pydantic-settings apscheduler
```

Или через `pip` из активированного окружения:

```powershell
pip install aiogram sqlalchemy aiosqlite pydantic-settings apscheduler
```

## Настройка `.env`

Скопируй `.env.example` в `.env` и заполни значения.

Пример:

```env
BOT_TOKEN=1234567890:ABCDEF_your_token
DATABASE_URL=sqlite+aiosqlite:///booking_bot.db
TIMEZONE=Asia/Yekaterinburg
SLOT_MINUTES=30
CANCEL_LIMIT_MINUTES=60
AUTO_CONFIRM_BOOKINGS=true
ADMIN_IDS=123456789,987654321
```

### Описание переменных

- `BOT_TOKEN` - токен Telegram-бота
- `DATABASE_URL` - строка подключения к БД
- `TIMEZONE` - часовой пояс проекта
- `SLOT_MINUTES` - шаг слотов в минутах
- `CANCEL_LIMIT_MINUTES` - за сколько минут до начала клиент еще может отменить запись
- `AUTO_CONFIRM_BOOKINGS` - автоподтверждение записи: `true` или `false`
- `ADMIN_IDS` - Telegram ID администраторов через запятую

## Как узнать свой Telegram ID

Можно использовать любого Telegram-бота для получения `user_id`, например `@userinfobot`, или временно вывести `message.from_user.id` в логах/обработчике.

## Запуск

Запуск из виртуального окружения:

```powershell
.\.venv\Scripts\python.exe main.py
```

Если всё настроено правильно, в логах появится что-то вроде:

```text
Project bootstrap completed. Bot token configured: yes
Starting Telegram polling
Start polling
```

## Первый запуск

После первого запуска проект:

- создает таблицы в базе данных
- добавляет базовые настройки
- создает основного мастера
- при необходимости мягко добавляет поле заметки к клиенту в таблицу `clients`
- запускает polling Telegram-бота
- запускает scheduler для напоминаний

## Что сделать после запуска

### 1. Открой бота в Telegram

Напиши:

```text
/start
```

### 2. Открой админку

Напиши:

```text
/admin
```

Если твой `Telegram ID` есть в `ADMIN_IDS`, откроется админ-панель.

### 3. Настрой базовые вещи

Рекомендуемый порядок:

1. `Админ -> Контакты мастера`
2. `Админ -> Настройки -> Изменить приветствие`
3. `Админ -> График -> Общие часы работы`
4. `Админ -> Услуги -> Добавить услугу`
5. при необходимости:
   - `График -> Исключения по датам`
   - `График -> Ручные блокировки`

## Сценарий клиента

Клиентский путь выглядит так:

1. нажимает `Записаться`
2. выбирает услугу
3. выбирает дату
4. выбирает время
5. подтверждает запись
6. получает статус записи
7. может открыть `Мои записи`
8. может отменить запись, если до начала осталось больше часа

## Сценарий администратора

### Услуги

Через раздел `Услуги` можно:

- добавить новую услугу
- открыть карточку услуги
- изменить название
- изменить цену
- изменить длительность
- изменить описание
- выключить или включить услугу
- удалить услугу

### График

Через раздел `График` можно:

- установить общие часы работы
- выбрать отдельную дату и сделать ее выходным
- задать особые часы на конкретную дату
- вручную заблокировать любой промежуток времени

### Записи

Через раздел `Записи` можно:

- смотреть ближайшие записи
- выбрать конкретную дату и посмотреть записи на этот день
- открыть карточку записи
- подтвердить запись, если автоподтверждение выключено
- отменить запись
- перенести запись на новый слот
- завершить запись
- сохранить заметку к клиенту

## Напоминания

Scheduler проверяет записи автоматически и отправляет клиентам:

- напоминание за `24 часа`
- напоминание за `1 час`

Отправленные уведомления логируются, чтобы не уходили повторно.

## Логика автоподтверждения

Если `AUTO_CONFIRM_BOOKINGS=true`:

- запись сразу создается как подтвержденная

Если `AUTO_CONFIRM_BOOKINGS=false`:

- запись создается в статусе ожидания
- админ подтверждает ее вручную через раздел `Записи`

## Контакты и ограничения Telegram

Telegram не отдает номер телефона пользователя автоматически.

Это значит:

- бот видит `telegram_id`
- бот видит `username`
- бот видит `first_name` и `last_name`
- номер телефона можно получить только если пользователь сам отправит контакт

Сейчас проект не запрашивает номер клиента принудительно.

## Типичные проблемы

### Бот не запускается

Проверь:

- заполнен ли `BOT_TOKEN`
- активировано ли виртуальное окружение
- установлены ли зависимости

### `/admin` пишет, что нет доступа

Проверь:

- правильно ли указан твой ID в `ADMIN_IDS`
- перезапущен ли бот после изменения `.env`

Пример:

```env
ADMIN_IDS=123456789
```
